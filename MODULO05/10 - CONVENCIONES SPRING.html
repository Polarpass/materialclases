Configuración de Proyectos Spring Boot

Guía completa para configurar proyectos en start.spring.io - Convenciones y buenas prácticas

¿Qué debemos ingresar en start.spring.io?

Configurar correctamente los campos Group, Artifact, Name y Description es fundamental para mantener consistencia y seguir las mejores prácticas en proyectos Java con Spring Boot.

Campos del Project Metadata

Group

Identificador único del grupo de proyectos

Convención: Usar el dominio inverso de la organización

Ejemplo: com.edumanage o edu.institucion

Importante: Mantener el mismo Group para todos los microservicios del sistema

Para EduManage: com.edumanage

Artifact

Nombre específico del microservicio

Convención: Nombre en minúsculas, sin espacios

Ejemplo: estudiantes-service o estudiantes-microservice

Recomendado: Usar sufijo "-service" para claridad

Para el microservicio: estudiantes-service

Name

Nombre descriptivo del proyecto

Convención: Usualmente igual que Artifact

Opcional: Puede ser más descriptivo si se desea

Propósito: Para documentación y reconocimiento

Ejemplo: estudiantes-service

Description

Breve descripción de la funcionalidad

Convención: Descripción clara y concisa

Ejemplo: Describe qué hace el microservicio

Propósito: Para documentación automática

Ejemplo: Microservicio para gestión centralizada de estudiantes, matrículas y antecedentes académicos

Package name

Estructura de paquetes Java

Convención: Se autogenera como: Group + Artifact

Ejemplo: com.edumanage.estudiantesservice

Importante: Spring Initializr lo genera automáticamente

Se genera: com.edumanage.estudiantesservice

Packaging

Tipo de empaquetado

Convención: Jar para microservicios

Alternativa: War (para aplicaciones web tradicionales)

Recomendación: Jar con Spring Boot Embedded Tomcat

Configuración: Jar ✓

Caso: Sistema de Gestión Educativa (EduManage)

Ejemplo práctico con 10 microservicios

Contexto del Sistema

La institución requiere una solución para mantener un registro estructurado, confiable y consultable de los estudiantes y su información académica asociada.

com.edumanage
├── estudiantes-service
├── profesores-service
├── cursos-service
├── calificaciones-service
└── ...

Estructura de Microservicios

1

estudiantes, matrículas, antecedentes

2

3

Cursos

cursos, asignaturas, prerequisitos

4

Calificaciones

evaluaciones, calificaciones, escalas

Ejemplo Concreto - Microservicio de Estudiantes

Project:
Language: Java
Spring Boot: 3.2.0 # Última versión estable

Project Metadata:
Group: com.edumanage
Artifact: estudiantes-service
Name: estudiantes-service
Description: Microservicio para gestión centralizada de estudiantes, matrículas y antecedentes académicos
Package name: com.edumanage.estudiantesservice # Autogenerado
Packaging: Jar
Java: 17 # Versión LTS recomendada

Resultado Esperado

  • Estructura de paquetes clara y organizada
  • Nombre descriptivo del microservicio
  • Consistencia con otros microservicios
  • Documentación automática generada

Buenas Prácticas y Recomendaciones

Nomenclatura

Usar nombres descriptivos en inglés, minúsculas y sin espacios

Consistencia

Mantener mismo Group para todos los microservicios del sistema

Versiones Java

Usar Java 17 o 21 (LTS - Long Term Support)

Spring Boot

Usar la última versión estable (actualmente 3.x.x)

Configuraciones Técnicas Recomendadas

Java Version

Recomendado: 17 o 21

Versiones LTS con soporte a largo plazo

Spring Boot

Recomendado: 3.2.x

Última versión estable con mejoras de seguridad

Packaging

Recomendado: Jar

Para microservicios con Tomcat embebido

Checklist para Estudiantes

Antes de generar tu proyecto en start.spring.io, verifica:

  • Group sigue el formato dominio inverso
  • Artifact usa minúsculas y sin espacios
  • Description es clara y descriptiva
  • Java versión es 17 o superior
  • Packaging es Jar para microservicios
  • Spring Boot es versión 3.x.x

Conclusión para Estudiantes

Puntos Clave

  • Group = Dominio inverso de la organización
  • Artifact = Nombre del microservicio
  • Name = Usualmente igual que Artifact
  • Description = Breve y descriptiva
  • Package name = Se autogenera (Group + Artifact)
  • Packaging = Jar para microservicios
